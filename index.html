🔢 Numerical Ability — Actual PYQ Types (2020–2025)

Percentages / Profit & Loss

Very High
PYQ Type — Appears in 90% of papers
If CP of 15 articles = SP of 12 articles, find the gain %.
→ Answer: 25%  |  Formula: (15–12)/12 × 100
Pattern: CP-SP-gain questions. Always set CP=1 for easy calculation.
PYQ Type — Compound Interest
A sum doubles in 5 years at compound interest. Find the rate.
→ Answer: ≈ 14.87% ≈ 15%  |  Use A = P(1+r)^t, put A=2P
Pattern: Doubling time → CI formula. Approximate, don't solve exactly.
PYQ Type — Discount
Man buys shirt at 25% discount on MP, sells at 20% profit on MP. MP = ₹400. Find SP.
→ CP = 400×0.75 = ₹300; SP = 400×1.20 = ₹480; But profit is on MP→ SP = ₹480
Pattern: Read carefully — profit basis (CP or MP) changes everything.

Time, Speed & Distance

Very High
PYQ Type — Relative Speed
A factory produces x units in 24 days. How many units in 15 days?
→ Rate = x/24/day → 15 × x/24 = 5x/8 units
Pattern: Direct proportion. Units proportional to days.
PYQ Type — Average Speed
Car travels 240km at 60 km/h for 4 hrs, then 40 km/h for 3 hrs. Average speed?
→ Total dist = 240+120=360km; Total time = 7hrs; Avg = 360/7 ≈ 51.4 km/h
Pattern: Always use Total Distance / Total Time. Never average the speeds directly.
PYQ Type — Two Bodies Meeting
A runs 10km while B runs only 9km. In a 100km race, by how much does A beat B?
→ When A finishes 100km, B has run 90km. A beats B by 10km.
Pattern: Race questions — find where B is when A finishes.

Permutations & Combinations

High
PYQ Type — Team Selection (very common)
Select a team of 11 from 5 men and 11 women, with no more than 3 men.
→ Cases: 0M+11W + 1M+10W + 2M+9W + 3M+8W = 1+55+550+1650 = 2256
Pattern: "No more than" = sum of all valid cases (0,1,2,3 men). Classic P&C trap.
PYQ Type — Digit Counting
How many 5s appear in numbers from 121 to 356?
→ Count 5 at units + tens + hundreds place across range = 51
Pattern: Break into ranges (121–200, 201–300, 301–356). Count by digit position.
PYQ Type — Series Sum (hard)
Find: 1(1!) + 2(2!) + 3(3!) + … + 2012(2012!)
→ n·n! = (n+1)! – n! → Sum telescopes → Answer: 2013! – 1
Pattern: Factorial series. Key trick: n·n! = (n+1)! – n!

Ratio, Mixtures & Alligation

High
PYQ Type — 3-Person Ratio
₹3000 split among A, B, C. A gets 2/3 of (B+C). C gets 1/3 of (A+B). Find C's share.
→ Set up equations: A = 2/3(3000-A) → A=1200; C = 1/3(3000-C) → C=750
Pattern: "A gets 2/3 of rest" — express in terms of total. Classic distribution trap.
PYQ Type — Distribution
If A:B = 2:3 and B:C = 4:5, find A:B:C
→ LCM B: A:B = 8:12, B:C = 12:15 → A:B:C = 8:12:15
Pattern: Make B's value equal (LCM method) then combine ratios.

🧠 Reasoning Ability — Actual PYQ Types (2020–2025)

Number & Letter Series

Very High
PYQ — Letter Series (appeared 2022, 2023, 2024)
Find the wrong term: CMQ, FPT, JTX, OYC, UFI
→ Answer: UFI  |  Gap between 1st letters: +3, +4, +5, +6 (not +5 again)
Pattern: Find the gap pattern. Check all three letters in each group separately.
PYQ — Weight/Rod Series Problem
O=2L; L=K+M; M=2K; N=3M; N=90kg. Find O.
→ N=3M=90 → M=30; K=15; L=45; O=2×45=90kg
Pattern: Start from known value and work backwards. Always start with most defined variable.

Blood Relations

Very High
PYQ — Multi-level Relation (very common format)
Karan is Ritu's son. Akshita is Ritu's daughter. What is Karan's wife to Akshita?
→ Karan = Akshita's brother → Karan's wife = Akshita's Sister-in-Law
Pattern: Draw a family tree. Male/female matters — mark gender for every person mentioned.
PYQ — Coded Relations
A $ B = A is father of B; A # B = A is sister of B; A @ B = A is mother of B. P # Q $ R @ S. How is P related to S?
→ P is sister of Q; Q is father of R; R is mother of S → P is grandaunt of S
Pattern: Decode each symbol, build chain. Draw diagram. Never solve in your head.

Coding-Decoding

Very High
PYQ — Letter Shift Coding
If MANGO = QERKS, then APPLE = ?
→ M+4=Q, A+4=E, N+4=R... each letter +4 → APPLE: A→E, P→T, P→T, L→P, E→I = ETTPI
Pattern: Find the shift value. Always check is it +n or -n or alternating.
PYQ — Word Position Coding
In a code, FREEZE = 3 and DOCTOR = 6. What is CHAIR?
→ Count unique vowels: FREEZE=1(E), DOCTOR=1(O); CHAIR=2(A,I) → answer 2
Pattern: Look for pattern (vowel count, letter count, unique letters, etc.)

Data Sufficiency & Statements

High
PYQ — Ranking / Ages
What is Priya's rank in class? (I) 15 students above her. (II) 8 students below her.
→ Stmt I alone: rank = 16; Stmt II alone: can't determine (don't know total); Answer: Only I sufficient
Pattern: Test each statement alone FIRST, then together. The answer options are always A/B/C/D/E format.
PYQ — Statement & Conclusion
Statement: All roses are flowers. Some flowers fade quickly. Conclusions: (I) Some roses fade quickly. (II) All flowers are roses.
→ Neither conclusion follows. "Some flowers" ≠ "Some roses" (roses are a subset)
Pattern: Use Venn diagrams. "All A are B" ≠ "All B are A". Common trap.

📝 Verbal Ability — Actual PYQ Types (2020–2025)

Grammar Error Spotting

Very High
PYQ — Subject-Verb Agreement (most common error type)
"Everyday life have become more complicated with the advancement in mobile technology." Find the error.
→ Error: "have" should be "has" (life = singular noun)
Pattern: Check if subject is singular/plural. "life", "team", "group" = singular verb "has/is/was".
PYQ — Tense Consistency
Combine: "I was at the fair. I got lost. I got scared."
→ Best: "I got lost at the fair and was scared." (all past tense, logical sequence)
Pattern: Best sentence = maintains tense, is concise, and logically sequential. Avoid "am/is" when context is past.

Reading Comprehension

High
RC Strategy (PYQ Analysis)
Passages are 150–250 words. 3–4 questions each. Topics: technology, environment, social issues.
→ Strategy: Read questions FIRST → Skim passage → Answer inference Qs last
Pattern: "Best title" = main idea. "Author's tone" = look for positive/negative/neutral adjectives. Inference = not stated directly.
Vocabulary in Context (PYQ)
"The scientist's hypothesis was tenuous at best." Closest meaning:
→ Tenuous = weak/flimsy. Not "strong" or "complex". Context clue: "at best" = barely valid.
Pattern: If unknown word — use surrounding context clues. "at best", "despite", "however" are strong clue signals.

⌨️ Programming Logic (MCQ) — Actual PYQ Types

Code Trace / Output MCQ

Very High
PYQ — Pointer & Array Output (C language)
int arr[] = {1,2,3,4,5}; int *p = arr; printf("%d", *(p+2));
→ Output: 3  |  p points to arr[0]; p+2 points to arr[2] = 3
Pattern: Pointer arithmetic. *(p+n) = arr[n]. Draw boxes for each array element.
PYQ — Loop Trace
int i=1; while(i<=5){ if(i%2==0) printf("%d ",i); i++; }
→ Output: 2 4
Pattern: Trace loops on paper. Write i value and condition check at each step.
PYQ — Scope / Variable
What is the output? int x=5; { int x=10; printf("%d",x); } printf("%d",x);
→ Output: 10 5  |  Inner block creates new scope. Outer x unchanged.
Pattern: Block scope in C. Inner { } creates new scope. Outer variable not modified.

Algorithm / Logic MCQ

High
PYQ — Time Complexity
What is the time complexity of binary search on a sorted array of n elements?
→ O(log n) — each step halves the search space
Pattern: Know Big-O for: Linear search=O(n), Binary=O(log n), Bubble sort=O(n²), Merge sort=O(n log n)
PYQ — Data Structure Choice
Which data structure is best for implementing a browser's Back button?
→ Stack (LIFO — last visited page is first to go back to)
Pattern: Stack=undo/back/parentheses. Queue=print spooler/BFS. Map=frequency count. Array=random access.
PYQ — SDLC / Pseudo-code
Which SDLC phase involves writing test cases before writing code?
→ Test-Driven Development (TDD) — Testing phase / design phase
Pattern: SDLC order: Requirement → Design → Implementation → Testing → Deployment → Maintenance

Real PYQ Coding Problems with Solutions

PYQ #1 — "Chocolate Factory" (Move Zeros to End)Python
# PYQ: N=8, arr=[4,5,0,1,9,0,5,0] → Output: [4,5,1,9,5,0,0,0]
def move_zeros(arr):
    pos = 0
    for i in range(len(arr)):
        if arr[i] != 0:
            arr[pos] = arr[i]
            pos += 1
    while pos < len(arr):
        arr[pos] = 0
        pos += 1
    return arr

n = int(input())
arr = [int(input()) for _ in range(n)]  # TCS takes 1 input per line
print(*move_zeros(arr))
PYQ #2 — Palindrome Check (String)Python
# Classic PYQ: Check if string is palindrome (ignore case, spaces)
s = input().strip().lower().replace(" ", "")
if s == s[::-1]:
    print("YES")
else:
    print("NO")

# Manual approach (safer for TCS compiler):
n = len(s)
is_pal = True
for i in range(n//2):
    if s[i] != s[n-1-i]:
        is_pal = False
        break
print("YES" if is_pal else "NO")
PYQ #3 — Find Second Largest ElementPython
# PYQ: Find 2nd largest without sorting (O(n) solution expected)
n = int(input())
arr = [int(input()) for _ in range(n)]

largest = second = float('-inf')
for num in arr:
    if num > largest:
        second = largest
        largest = num
    elif num > second and num != largest:
        second = num

if second == float('-inf'):
    print("No second largest")
else:
    print(second)
PYQ #4 — Odd-Occurring Element (XOR trick)Python
# PYQ: Array where every element appears even times EXCEPT one
# Input: 5 2 2 3 1 1 → Output: 3
n = int(input())
result = 0
for _ in range(n):
    result ^= int(input())  # XOR: a^a=0, a^0=a → only odd one remains
print(result)

# Why XOR works: 2^2=0, 1^1=0, 0^3=3 → result=3 ✓
⚡
TCS Input Format Warning: TCS compiler often takes one input per line (not space-separated). Always write int(input()) in a loop, not list(map(int, input().split())). Check the problem statement input format carefully before writing any code.

🔴 OOPS — Most Asked (100% appearance)

What are the 4 pillars of OOPS? Explain each with a real-world example.
Encapsulation: Capsule with multiple medicines (data + methods in one class). Inheritance: Child inherits parent properties (IS-A). Polymorphism: Same function, different behaviour (overloading/overriding). Abstraction: Mobile phone — you use it without knowing internal signal processing.
What is the difference between Method Overloading vs Overriding?
Overloading: Same method name, different parameters — compile-time polymorphism. Overriding: Child class redefines parent's method — runtime polymorphism.
Is C++ a pure OOP language? Why/Why not?
No. Pure OOP means ALL code inside classes, no primitive types. C++ allows functions outside classes and uses primitive data types (int, float).
Explain the 5 types of inheritance with examples.
Single, Multiple, Multilevel, Hierarchical, Hybrid. Example: Animal→Dog (single), Dog+Cat→Pet (multiple).
What is an abstract class vs interface?
Abstract class: Can have both abstract and non-abstract methods; single inheritance only. Interface: All methods abstract (by default); supports multiple inheritance.
What is a constructor? Can a constructor be private?
Constructor initializes object. Yes, private constructor → Singleton design pattern (only one instance allowed).

🔵 DBMS — Most Asked (95% appearance)

What is a DBMS? Name the types of databases.
Computerized data-keeping system. Types: Relational (MySQL, PostgreSQL), Document (MongoDB), Key-Value (Redis), Graph (Neo4j), In-memory (Redis).
Explain INNER JOIN, LEFT JOIN, RIGHT JOIN with examples.
INNER: Only matching rows in both tables. LEFT: All rows from left + matching right (NULLs where no match). RIGHT: All rows from right + matching left.
What is normalization? Explain 1NF, 2NF, 3NF.
1NF: No repeating groups, atomic values. 2NF: 1NF + no partial dependency on composite key. 3NF: 2NF + no transitive dependency.
What is a Primary Key vs Foreign Key?
Primary Key: Uniquely identifies a row, no NULLs. Foreign Key: References PK of another table — maintains referential integrity.
What are SQL Triggers?
Automatic actions executed when INSERT/UPDATE/DELETE happens on a table. Used to enforce business rules or maintain audit logs.
What is indexing? Why is it used?
Indexing creates a data structure (B-tree usually) for faster data retrieval. Without index: O(n) scan. With index: O(log n). Trade-off: Slower writes, more storage.

🟢 OS — Highly Asked (85% appearance)

What is the difference between a Process and a Thread?
Process: Independent program in execution, own memory space. Thread: Lightweight sub-process, shares memory with parent. Threads faster to create, communicate easier.
What is paging? Explain pages and frames.
Memory management technique. RAM divided into fixed-size frames. Process divided into same-size pages. OS maps pages to frames via page table. Eliminates external fragmentation.
What is a deadlock? What are the 4 conditions?
Processes blocked waiting for each other's resources forever. 4 conditions (Coffman): Mutual Exclusion, Hold & Wait, No Preemption, Circular Wait. Prevention: Eliminate any one condition.
What are CPU scheduling algorithms?
FCFS (First Come First Serve), SJF (Shortest Job First), Priority Scheduling, Round Robin (time quantum). Round Robin is most fair for time-sharing systems.
What is virtual memory?
Technique that uses HDD as extension of RAM. Allows programs larger than physical RAM to run. Uses paging/segmentation. Slower than physical RAM access.

🟡 Computer Networks (75% appearance)

Explain the 7 layers of the OSI model.
Physical→Data Link→Network→Transport→Session→Presentation→Application. Mnemonic: "Please Do Not Throw Sausage Pizza Away" (bottom to top).
TCP vs UDP — when to use each?
TCP: Connection-oriented, reliable, ordered, slower (HTTP, FTP, email). UDP: Connectionless, unreliable, faster (video streaming, DNS, gaming).
What is HTTP vs HTTPS?
HTTP: Plain text, port 80, no encryption. HTTPS: SSL/TLS encrypted, port 443. All TCS Digital projects must use HTTPS — mention this from your Next.js projects.
What is DNS and how does it work?
Domain Name System: Translates human-readable domain (google.com) to IP addresses. Hierarchical: Root servers → TLD servers → Authoritative servers. Uses UDP.
What is an IP address? IPv4 vs IPv6?
IPv4: 32-bit, 4.3 billion addresses (e.g., 192.168.1.1). IPv6: 128-bit, virtually unlimited. IPv4 exhaustion drove IPv6 adoption.

🟠 Live Coding (Digital/Prime Only)

Write code to reverse a string without built-in functions.
Two pointer: l=0, r=len-1; while l<r: swap s[l] and s[r], l++, r--. O(n) time O(1) space.
Print this pattern: * → ** → *** → ****
for i in range(1,N+1): print('*'*i). Know both star and number pyramid patterns.
Check if string has balanced parentheses.
Stack approach: push ( on open, pop and check on close. If stack empty at end = balanced. O(n).
Find duplicate elements in an array.
Use set: seen=set(); for num in arr: if num in seen: print(num); else: seen.add(num). O(n) time O(n) space.
Write SQL: Find employees with salary > department average.
SELECT * FROM employees e WHERE salary > (SELECT AVG(salary) FROM employees WHERE dept_id = e.dept_id);
ROUND 3 — MANAGERIAL INTERVIEW

Managerial Round
Question Bank

20–30 minutes. Tests leadership, problem-solving, situational judgement. Uses STAR format (Situation → Task → Action → Result). Sourced from TCS NQT 2022–2025 interview experiences.

STAR method: Every behavioral answer = Situation (context) + Task (your role) + Action (what you specifically did) + Result (outcome + what you learned). Keep each answer under 2 minutes. Always end on a learning or positive outcome.

Tell me about a time you failed. What did you learn?
Pick a real but recoverable story. End with: "I learned to [test more rigorously / communicate earlier / plan better]". Never say "I've never failed."
How do you prioritize when you have multiple tasks?
Mention: Urgent vs Important matrix (Eisenhower), breaking tasks into sub-tasks, communicating with stakeholders about realistic timelines.
Where do you see yourself in 3 years at TCS?
"Contributing to TCS's AI/ML practice, growing from an engineer to a senior developer, and potentially leading a small module. I'd like to take TCS certifications in cloud and AI to grow technically."
Are you comfortable with relocation? Rotational shifts?
ALWAYS say YES to both — no exceptions. This is not negotiable in the managerial round. Any hesitation is noted.
How do you keep up with technology changes?
Mention GeeksforGeeks writing (20+ articles = you teach = you learn deeply), open source (GSSoC), personal projects experimenting with new tech (Gemini AI, Supabase).

Situational / Judgment Questions

"Your code breaks in production at 2 AM. What do you do?"
1. Rollback to last working version immediately. 2. Notify team lead. 3. Reproduce the bug in staging. 4. Fix and test thoroughly. 5. Deploy with monitoring. 6. Write a post-mortem.
"A senior colleague tells you to implement something that you think is wrong. What do you do?"
1. Ask for clarification respectfully. 2. Present your concern with data/examples. 3. Escalate if still unclear. 4. If still asked to proceed — document your concern and proceed professionally.
"How would you handle a client who is unhappy with your deliverable?"
Listen actively. Acknowledge the concern. Clarify what was expected vs delivered. Present a plan to fix it with a timeline. Never be defensive.
"A team member is not contributing equally. What do you do?"
First check privately if they have blockers. Offer help. If continues, involve team lead. Document contribution issues if needed. TCS values team harmony — empathy first.

Formula Sheet
& Quick Reference

The only formulas you need memorized cold for NQT. These appear in 80%+ of numerical questions across PYQs.

Profit & Loss
Gain% = (SP–CP)/CP × 100
CP = 100/(100+P%) × SP
CP of 15 = SP of 12 → Gain = 3/12×100 = 25%
Compound Interest
A = P(1 + r/100)^t
Doubles in t yrs: 2 = (1+r)^t
Doubling in 5 yrs ≈ 15% rate (approx)
Time Speed Distance
Speed = Distance / Time
Avg speed = Total D / Total T
NEVER average the two speeds directly
Time & Work
If A takes a days, rate = 1/a/day
Together = 1/(1/a + 1/b) days
A=10d, B=15d → Together = 6 days
Percentages
x% of y = y% of x (trick!)
% change = (new–old)/old × 100
16% of 25 = 25% of 16 = 4
Permutations
nPr = n!/(n–r)!
nCr = n!/(r!(n–r)!)
5C2 = 5!/(2!×3!) = 10
Series Sum Tricks
Sum 1 to n = n(n+1)/2
n·n! = (n+1)! – n!
1(1!)+2(2!)+...+n(n!) = (n+1)!–1
Mixtures / Alligation
Cheaper:Dearer = (D–M):(M–C)
where M = mean price
Mix ₹30 & ₹50 to get ₹40: ratio = 1:1
Ratio Combining
A:B = 2:3, B:C = 4:5
Make B same: A:B = 8:12, B:C = 12:15
A:B:C = 8:12:15
Big-O Cheat Sheet
Linear search: O(n)
Binary search: O(log n)
Bubble/Selection sort: O(n²) · Merge: O(n log n)
XOR Properties
a ^ a = 0, a ^ 0 = a
XOR all elements → odd-occurring remains
2^2^3^1^1 = 0^3^0 = 3
Armstrong Number
Sum of (each digit)^n = number
153: 1³+5³+3³ = 1+125+27 = 153 ✓
Others: 370, 371, 407 (3-digit)
